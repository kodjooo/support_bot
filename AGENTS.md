# AGENTS.md — инструкции для AI-агента

Основные правила  описаны в 'docs/arch-rules.md'
Бизнес требования к сервису описаны в 'docs/requirements.md'

При изменении проекта обязательно нужно вносить описание архитектуры в документ 'docs/architecture.md' использовать его в работе. Также нужно актуализировать '.env.example', '.env' файлы и тесты в папке 'tests'

После реализации каждого этапа плана из файла 'docs/plan.md' нужно обязательно отмечать его там (писать 'выполнено'), как выполненный, но не удалять

В .env файл пиши комментарии, какой доступ и где нужно найти.

В GITHUB не нужно отправлять .env файл

По завершению работ пушь изменения в ветку main в гитхаб в этот репозиторий https://github.com/kodjooo/support_bot.git, инициируй его, если это еще не сделано. Информацию по разворачиванию проекта на удаленном сервере нужно писать в readme.md.

Для этого репозитория используй `origin` по HTTPS и выполняй push через HTTPS-аутентификацию, а не через SSH-ключ.

Если тебе нужна какая-то документация, сначала попробуй найти ее в MCP Context7 и если ее там нет - попробуй найти ее в интернете.

Если тебе нужно протестировать что-то во фронтенде - сделай это через MCP Playwright.
